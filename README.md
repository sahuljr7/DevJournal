# DevJournal: Technical Ledger System

## Purpose
DevJournal is a high-fidelity work management and documentation platform designed for engineers and researchers. It bridges the gap between structured project management (Kanban) and detailed technical logging (Daily Journals). Unlike standard task managers, DevJournal treats every status update as a piece of technical evidence, allowing for rigorous tracking of technical assets, source files, and development logic.

## Core Functionality
- **Dual-Pane Record Management**: A sophisticated Kanban board transitions into a deep-dive editorial ledger for specific tasks.
- **Rich Technical Ledger**: Support for Markdown-based work logs with integrated technical source consignment.
- **Automated Version Control**: All attachments consigned to the ledger are implicitly versioned, allowing one-click restoration of historical file states.
- **In-situ PDF & Image Inspection**: High-fidelity in-line previews for PDFs and images directly within the documentation flow.
- **Intelligence Layer**: Integrated AI synthesis for generating executive summaries from long-running development ledgers.
- **Robust Export Suite**: 
  - **Executive PDF Reports**: Generates professional task summaries including full audit trails.
  - **Batch Source Archiving**: One-click ZIP bundling of all text entries and historical attachments.

## How to Interact
1. **Initiate Records**: Create new cards on the Kanban board for primary workstreams.
2. **Consign Logic**: Use the "New Entry" button in the record view to draft technical logs.
3. **Attach Evidence**: Drag and drop technical source files (PDFs, Images) into the editor.
4. **Iterate**: Update files by using the versioning system to preserve history.
5. **Synthesize**: Use the "Summarize" button to get an AI-generated overview of complex technical progress.
6. **Report**: Generate PDF reports or ZIP archives for external stakeholders or stakeholders.
